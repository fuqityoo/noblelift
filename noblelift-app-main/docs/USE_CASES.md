# Сценарии использования (Use Cases) — CRM Noblelift

## 1. Аутентификация

| ID | Сценарий | Шаги | Ожидаемый результат |
|----|----------|------|--------------------|
| UC-1.1 | Вход в систему | 1. Открыть приложение 2. Ввести email и пароль 3. Нажать «Войти» | Пользователь авторизован, отображается главный экран с вкладками |
| UC-1.2 | Неверные учётные данные | 1. Ввести неверный email/пароль 2. Нажать «Войти» | Сообщение «Неверный email или пароль» |
| UC-1.3 | Выход | 1. Выйти из приложения (если реализовано в профиле) | Токены сброшены, показан экран входа |

---

## 2. Задачи (Мои задачи)

| ID | Сценарий | Шаги | Ожидаемый результат |
|----|----------|------|--------------------|
| UC-2.1 | Просмотр своих задач | 1. Войти 2. Открыть вкладку «Задачи» | Список задач, назначенных текущему пользователю (не завершённых) |
| UC-2.2 | Поиск по задачам | 1. На вкладке «Задачи» ввести текст в поле поиска | Список отфильтрован по заголовку/содержанию |
| UC-2.3 | Отметить задачу «Готово» | 1. Открыть задачу 2. Нажать «Готово» | Статус задачи = «Завершена», задача обновлена на бэкенде |
| UC-2.4 | Развернуть/свернуть описание | 1. Нажать «Показать полностью» / «Свернуть» под длинным описанием | Описание разворачивается или сворачивается |

---

## 3. Доступные задачи

| ID | Сценарий | Шаги | Ожидаемый результат |
|----|----------|------|--------------------|
| UC-3.1 | Просмотр доступных задач | 1. Открыть вкладку «Доступные» | Список общих задач без исполнителя |
| UC-3.2 | Взять задачу | 1. Нажать «Взять» на карточке задачи | Задача назначена на текущего пользователя, статус «В работе», исчезает из «Доступных» |

---

## 4. Создание задачи

| ID | Сценарий | Шаги | Ожидаемый результат |
|----|----------|------|--------------------|
| UC-4.1 | Создать личную задачу | 1. Вкладка «Создать» 2. Тип «Личная» 3. Заполнить заголовок, содержание, срок, приоритет 4. «Создать» | Задача создана и назначена на текущего пользователя |
| UC-4.2 | Создать общую задачу (супер-админ) | 1. Тип «Общая» 2. Заполнить поля 3. «Создать» | Задача создана как общая, появляется в «Доступные» |
| UC-4.3 | Валидация | 1. Оставить заголовок или содержание пустым 2. «Создать» | Сообщение об ошибке, задача не создаётся |
| UC-4.4 | Прикрепить файл (Web) | 1. Выбрать файл 2. Создать задачу | Файл загружается на сервер и привязывается к задаче |

---

## 5. Профиль

| ID | Сценарий | Шаги | Ожидаемый результат |
|----|----------|------|--------------------|
| UC-5.1 | Просмотр профиля | 1. Вкладка «Профиль» | Отображаются ФИО, должность, контакты, статус |
| UC-5.2 | Редактирование профиля | 1. Изменить ФИО/должность/телефон 2. «Сохранить» | Данные сохранены на бэкенде |
| UC-5.3 | Смена статуса | 1. Нажать поле «Статус» 2. Выбрать статус в модалке 3. «Сохранить» | Статус обновлён |

---

## 6. Автопарк

| ID | Сценарий | Шаги | Ожидаемый результат |
|----|----------|------|--------------------|
| UC-6.1 | Просмотр списка ТС | 1. Вкладка «Автопарк» | Список машин с маркой, моделью, цветом, госномером, статусом |
| UC-6.2 | Взять машину | 1. Нажать «Взять» у свободной машины | Машина в статусе «На руках у вас» |
| UC-6.3 | Отдать машину | 1. Нажать «Отдать» у взятой машины | Машина снова «Свободна» |
| UC-6.4 | Добавить машину (менеджер/супер-админ) | 1. Заполнить марку, модель, цвет, госномер 2. «Добавить» | Машина создана и отображается в списке |
| UC-6.5 | Удалить машину | 1. Нажать удаление у машины (если есть право) | Зависит от бэкенда: DELETE может быть не реализован |

---

## 7. Справочник и архив

| ID | Сценарий | Шаги | Ожидаемый результат |
|----|----------|------|--------------------|
| UC-7.1 | Справочник | 1. Вкладка «Справочник» | Отображаются моковые документы и коллеги |
| UC-7.2 | Завершённые | 1. Вкладка «Завершённые» | Сообщение о пустом списке или список из state.archivedTasks (при интеграции с API архива) |

## 8. Адаптивность и навигация

| ID | Сценарий | Шаги | Ожидаемый результат |
|----|----------|------|--------------------|
| UC-8.1 | Мобильный вид | 1. Открыть приложение на узком экране (< 1024px) | Нижняя панель вкладок, горизонтальный скролл вкладок при необходимости |
| UC-8.2 | Десктоп | 1. Широкий экран (≥ 1024px) | Боковое меню, контент справа |
| UC-8.3 | Переключение вкладок | 1. Нажимать вкладки | Меняется экран, заголовок в шапке обновляется |
