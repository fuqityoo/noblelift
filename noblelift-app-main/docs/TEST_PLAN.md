# План тестирования

## Запуск приложения

```bash
cd work/noblelift-app-main
npm install   # при необходимости
npm run web   # веб
# или
npm start     # Expo, затем w для web / a для Android / i для iOS
```

Убедитесь, что бэкенд доступен (по умолчанию `http://26.102.245.102:8000/api/v1` или задайте `EXPO_PUBLIC_API_BASE_URL`).

## Исправленные баги (для регрессии)

1. **Задачи — список «Мои»**: запрос к API с `assignee_id` (snake_case), не `assigneeId`.
2. **Доступные задачи**: запрос к `/tasks/available`, не к `/tasks?assigneeId=`.
3. **Смена статуса задачи**: PATCH `/tasks/{id}` с `statusCode`, не POST `/tasks/{id}/set-status`.
4. **Назначение исполнителя**: PATCH `/tasks/{id}` с `assigneeId`, не POST `set-assignee`.
5. **Взять/вернуть общую задачу**: POST `/tasks/{id}/take` и `/tasks/{id}/release`.
6. **Создание задачи**: тип личной — `regular`, поле `content`, не `description`.
7. **Автопарк — список**: поля `brand`, `number` (госномер), не `name`/`plate`.
8. **Автопарк — создание**: тело `{ number, brand, model, color }`, не `name`/`plate`.
9. **Отображение задач**: поддержка и API-полей (statusCode, dueDate), и модели Task (status, due).
10. **Мобильная навигация**: нижние вкладки с горизонтальным скроллом, адаптивная шапка.

## Рекомендуемый порядок проверки

1. Вход → главный экран.
2. Задачи: список, поиск, кнопка «Готово».
3. Доступные: список, «Взять».
4. Создать задачу: личная с обязательными полями.
5. Профиль: просмотр, смена статуса, сохранение.
6. Автопарк: список, при наличии прав — взять/отдать, создание машины.
7. Узкое окно: вкладки скроллятся, контент не обрезается.

См. также [USE_CASES.md](./USE_CASES.md) для полного списка сценариев.
